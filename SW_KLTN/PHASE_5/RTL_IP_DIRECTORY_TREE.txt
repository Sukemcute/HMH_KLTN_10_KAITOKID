PHASE 3 — CÂY THƯ MỤC RTL CHO MỘT IP TÍNH TOÁN (YOLOv10n INT8 Accelerator V2)
Tham chiếu: RTL_MODULE_SPEC.md, HW_ARCHITECTURE_V2 (PHASE 0/1/2)
================================================================================

Nguyên tắc thiết kế (một IP duy nhất):
  • Một luồng dữ liệu thống nhất: DDR ↔ tensor_dma ↔ (SC×4) ↔ subcluster ×4
  • Mỗi subcluster = tile_fsm + shadow + GLB + addr_gen + router + window_gen
                 + pe_cluster + ppu + swizzle_engine (KHÔNG nhân đôi conv bằng behavioral)
  • Điều khiển tập trung: controller_system (CSR, desc_fetch, barrier, global_scheduler)
  • SoC boundary: accel_top (AXI-Lite + một cổng AXI4 256b master mux từ 4 SC + ctrl)

--------------------------------------------------------------------------------
GHI CHÚ TÍCH HỢP (một IP — theo spec, chưa đủ trong code hiện tại)
--------------------------------------------------------------------------------

  • subcluster_wrapper.sv: cần thay thế nhánh behavioral conv/PPU bằng nối dây:
      glb_* + addr_gen_* → router_cluster → window_gen → pe_cluster → ppu →
      swizzle_engine → handshake ext_* với tensor_dma (supercluster_wrapper).
  • global_scheduler: bổ sung inference_complete khi toàn bộ tile + DMA hoàn tất
    (không chỉ fetch_all_done), để STATUS/IRQ phản ánh đúng IP.
  • accel_top: giữ một slave AXI-Lite + một master AXI4 mux; đảm bảo AR/R/AW/W/B
    grant khớp từng chu kỳ handshake (đã có phần serialize 4 SC).
